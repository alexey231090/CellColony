# Правила проекта CellColony

## 1. Структура проекта
- `scenes/` — `.tscn` файлы (с вложенными папками по объектам).
- `scripts/` — `.gd` файлы (разделены по логике: `core`, `ai`, `resources`).
- `assets/` — Внешние ресурсы.
- `HUD/UI` — Всегда размещаются в `HUDLayer` (CanvasLayer с `layer = 100`), чтобы элементы оставались зафиксированными на экране.


## 2. Работа со сценами (The Script Attachment Rule)
- **ЗАПРЕЩЕНО:** Прямая вставка пути к скрипту в ноду в `.tscn`.
- **ОБЯЗАТЕЛЬНО:** Использование `[ext_resource]` и `ExtResource(ID)`.
- **Порядок инициализации:** При спавне нод из кода (`instantiate()`), основные свойства (`owner_type`, `position`, `stats`) ДОЛЖНЫ быть установлены ДО вызова `add_child()`. Это гарантирует корректную работу `_ready()` (группы, визуализация).

## 3. Код и типизация (GDScript 2.0)
- **Строгая типизация:** Все аргументы и возвращаемые значения функций должны быть типизированы.
- **Node-модули:** Сложную логику выносить в дочерние ноды (ShooterModule, HealthModule и др.).

## 4. Специфические правила CellColony
- **Параметры:** Все числовые лимиты (энергия, скорость, урон) должны браться из ресурса `CellStats`. Запрещено хардкодить числа в логике.
- **Фракции и Группы (OwnerType):** 
  - Все клетки обязаны быть в группе `cells`.
  - Динамические группы фракций: `player_cells`, `enemy_red_cells`, `enemy_green_cells`, `enemy_yellow_cells`.
  - Переключение фракций выполняется через `_capture(new_owner: OwnerType)` с обновлением групп и ОБЯЗАТЕЛЬНЫМ сбросом всех временных баффов (щиты, множители).
- **Мир и ИИ:**
  - Генерация мира (`main.gd`) обязана учитывать 4 базы по углам.
  - ИИ (`AIFactionManager`) использует систему весов (скоринг) для выбора целей по всей карте.
  - Режим наблюдателя активируется автоматически в `CameraFollow.gd`, когда группа `player_cells` пуста.
  - **Безопасность ИИ:** Поиск своих клеток в `AIFactionManager` должен иметь запасной вариант (fallback) через фильтрацию группы `cells` по `owner_type`, если фракционная группа пуста.
- **Безопасность:** Перед работой с `target_node` (узлом-целью) всегда использовать `is_instance_valid(target_node)`.
- **Стрельба:** Снаряды должны пролетать сквозь союзников, если они не являются целью (учитывать 4 фракции).
- **Взаимодействие систем:** 
  - Связь UI и кода должна идти через группы. Пример: Поиск `SelectionManager` через группу `selection_manager` в кнопке перка.
  - Состояния выбора (например, активация перка) должны храниться в `SelectionManager.gd` и блокировать стандартное поведение клика.
